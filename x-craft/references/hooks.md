# Hook Formulas and Examples

The first line of any tweet or thread is the only line that matters initially. It determines whether the algorithm shows the post to more people, whether someone stops scrolling, and whether they read what follows.

**Rule:** The hook must work as a standalone sentence. Someone reading only the first line should feel something — curiosity, surprise, recognition, or the urge to prove it wrong.

---

## The Five Hook Types

### 1. Number Hook

Lead with a specific number. Numbers create specificity and signal that the content is concrete, not vague.

**Formula:** `[Specific number] + [surprising or non-obvious thing]`

**Why it works:** Numbers stop the scroll because they promise precision. A vague claim creates no tension. A specific claim invites verification.

**Worked examples:**

```
37 people signed up on day one. I hadn't launched yet.
```
_(BIP — the paradox of signups before launch creates curiosity)_

```
3 lines of code replaced a service we were paying $400/month for.
```
_(Technical — the contrast between size and cost is the hook)_

```
$0 to $4,200 MRR. Here's every mistake we made getting there.
```
_(BIP — the number earns trust, "every mistake" earns the read)_

```
I've written 847 cold emails. 9 of them worked. Here's what was different.
```
_(Lesson — specificity of failure makes the win credible)_

```
11 agents. 1 pipeline. Ships 40 posts a day without human input.
```
_(Technical / product announcement — three numbers build the picture fast)_

---

### 2. Contrast Hook

Put two things in opposition. The tension between them is the hook.

**Formula:** `[Thing A] vs [Thing B]` or `[Expected thing] / [Unexpected reality]`

**Why it works:** Contrast forces the brain to reconcile two states. That cognitive friction keeps people reading to resolve it.

**Worked examples:**

```
Most people optimise for more users. The best operators optimise for fewer, better ones.
```
_(Opinion — contradicts the growth-at-all-costs default)_

```
We got 10,000 impressions this week. Made $0 from them. Here's why that's fine.
```
_(BIP — the apparent failure that isn't reframes the expectation)_

```
The feature took 2 hours to build. The decision about whether to build it took 3 weeks.
```
_(Lesson — exposes where the real work was)_

```
Everyone's talking about AI replacing developers. Nobody's talking about what developers are now free to do.
```
_(Opinion / hot take — same event, two framings)_

```
I used to write documentation nobody read. Then I wrote documentation that spread without me sharing it. One difference.
```
_(Educational — transformation with one promised variable)_

---

### 3. Question Hook

Ask a question the reader hasn't thought to ask themselves.

**Formula:** A question that the reader immediately wants the answer to — not a rhetorical question, not a yes/no question.

**Why it works:** Questions create an open loop. The brain wants closure. It reads on to close the loop.

**Warning:** Weak question hooks ("Have you ever wondered...?", "What if I told you...?") are overused and signal low effort. The question must be genuinely non-obvious.

**Worked examples:**

```
Why do some landing pages convert at 12% when the industry average is 2.3%? It's rarely the copy.
```
_(Educational — poses the question and immediately challenges the assumed answer)_

```
What happens to your MRR when you stop discounting? We found out.
```
_(BIP — implies a result without revealing it)_

```
Why do junior developers push to production on Friday afternoons? Because nobody told them not to.
```
_(Opinion / lesson — the question contains the answer as a reveal)_

```
What would you change if growth stalled for 90 days and you couldn't add features?
```
_(Opinion — forces the reader to apply it to themselves)_

---

### 4. Bold Statement Hook

State a position directly. No hedging, no qualification in the first line.

**Formula:** `[Definitive claim] — ideally one that challenges a common belief`

**Why it works:** Bold statements divide people. Some agree and want to share it. Some disagree and want to reply. Both are engagement. Hedged statements create neither response.

**Warning:** The post must back the statement up. A bold hook with a weak body destroys credibility. Bold hook + evidence = strong post. Bold hook + nothing = noise.

**Worked examples:**

```
Most developer tools are built for developers who want to feel smart, not solve problems faster.
```
_(Opinion — likely to generate replies from both camps)_

```
Your onboarding is not too long. It's not engaging enough to justify its length.
```
_(Opinion — reframes the standard advice without dismissing the underlying problem)_

```
A/B testing is a way to feel rigorous while avoiding hard decisions.
```
_(Hot take — extreme enough to provoke, defensible enough to hold)_

```
Email is still the highest-ROI channel in 2026. Anyone who says otherwise is selling you something else.
```
_(Hot take — stakes out a clear position and points at the motivation to disagree)_

```
The best thing you can do for your codebase is delete code.
```
_(Technical opinion — short, defensible, invites discussion)_

---

### 5. Story Hook

Open with a moment — a specific scene, action, or event. Not "I've been thinking about..." but an actual thing that happened.

**Formula:** `[Specific moment or action] + [why it matters or what it revealed]`

**Why it works:** Narrative creates a commitment device. Once the reader is in the story, they want to know how it ends.

**Warning:** Story hooks need to pay off. If the story is the whole post, it needs a strong ending. If it's the lead-in to a lesson or insight, the lesson must land clearly.

**Worked examples:**

```
A user emailed me at 2am to say our app saved their relationship. I didn't expect that to be in the feedback.
```
_(BIP / product — specific, human, creates curiosity about what the app does and what "saved" means)_

```
I shipped a feature nobody asked for. 40% of users turned it on in week one.
```
_(BIP / product announcement — the paradox earns the explanation)_

```
We interviewed 50 churned users. They all said the same thing. We'd missed it for two years.
```
_(Lesson — the scale of the miss makes the lesson feel important)_

```
Six months ago I rewrote our auth from scratch. Last week I reverted every line.
```
_(Technical / lesson — the completed arc told in two sentences creates maximum tension)_

```
I gave a user a refund they didn't ask for. They tripled their spend the next month.
```
_(Lesson — counter-intuitive outcome from a specific action)_

## Hook Selection by Content Type

| Content Type | Best Hook Types | Avoid |
|---|---|---|
| Build in Public | Number, Story | Bold statement (can read as boasting without receipts) |
| Technical deep dive | Number, Bold statement, Contrast | Question (can feel vague in technical context) |
| Lesson / insight | Story, Contrast, Number | Bold statement (lessons land better as discoveries, not proclamations) |
| Opinion / hot take | Bold statement, Contrast | Story (stories slow down hot takes) |
| Educational / save bait | Number, Question | Story (delays the value delivery) |
| Product announcement | Number, Story, Contrast | Bold statement (can read as hype) |

## Hook Anti-Patterns

Patterns that kill reach before the algorithm even looks at the body:

**Opening with "I"**
```
I've been thinking about this for a while...
I wanted to share something...
I've been building in public for 6 months...
```
These signal the post is about the writer, not the reader. Readers filter out posts that don't offer them immediate value.

**Vague openers**
```
Something interesting happened today.
Here's a thread on productivity.
Hot take incoming:
```
Vague openers create no tension. "Something interesting happened" — the reader doesn't yet know if it's interesting to them.

**Announcing what the post is**
```
Thread: everything I learned about retention
A quick thought on pricing
Some reflections on year one
```
Meta-commentary about the post delays the post. Start with the insight, not the label.

**Padding before the point**
```
So I've been building this project for about 6 months now, and I've learned a lot along the way...
```
Bury the lead and lose the reader. The learning goes in the first line. The 6 months of context comes second if needed.

## Testing a Hook

Before finalising, run through this checklist:

- Does it contain a number, a contrast, a question, a bold claim, or a specific moment?
- Could someone read only this line and feel something — curiosity, surprise, or the need to respond?
- Is it free of "I", "We", "Here's a thread", and vague openers?
- Would it make sense as a standalone tweet with no context?
- Is it specific enough to be credible?

If the answer to any of these is no, rewrite the first line before touching anything else.
